Salone Learn Hub (SLH)Salone Learn Hub (SLH) is a digital school management and e-learning platform designed as a Digital Public Good to improve educational outcomes in Sierra Leone. Developed using Agile software development methodologies, SLH addresses critical regional education challenges such as overcrowded classrooms, learning material shortages, poor student tracking, and urban-rural educational disparities.📖 Table of ContentsKey FeaturesSystem ArchitectureFigma PrototypeProject Timeline & PlanningSDG AlignmentLicense✨ Key FeaturesThe platform provides dedicated entry points and role-based workflows for four target user roles:Student (Learner App): Access to sign-up/login, dashboards, class schedules, grade reports ("My Grades"), assignments, e-library resources, and real-time notifications.Teacher (Staff Portal): Tools to manage class schedules, input grades, generate report cards, and execute the "Mark Attendance" feature.Administrator (Staff Portal): School registration forms (Form 1 & Form 2), search functionalities for student/teacher records, and platform governance.Parent/Guardian (Parent Portal): Connect directly with a child’s profile to receive automatic attendance alerts (e.g., student marked absent), view progress metrics, and view report cards.Public Access (Unauthenticated): Immediate access to the platform splash screen, "About Us" page, and school announcements for prospective parents or community members.🛠️ System ArchitectureThe SLH ecosystem uses a modular, scalable four-layer decoupled architecture:┌─────────────────────────────────────────────────────────────┐

Presentation Layer: Houses the UI entry points mocked in Figma (Learner App, Staff Portal, Public Access).Communication Layer (API Gateway): A unified gateway handling authentication tokens, session validation, and strict role-based access control (RBAC).Application/Business Logic Layer: Powered by four independent services (Authentication, Student Management, Class/Grading, and Notifications) that communicate asynchronously via the database and gateway.Data Layer & External Integrations: Features a single shared central database ("source of truth") paired with third-party SMS/Push providers and digital e-libraries.🎨 Figma PrototypeThe UX architecture has been completely mapped out through 21 interactive screens mapping the end-to-end flows for onboarding, administration, and grading.Figma Design Link: Access Figma Prototype📅 Project Timeline & PlanningThe project was executed by a student collective over a fixed academic 72-day schedule (Weeks 4–11) using an incremental Agile sprint-delivery system.Activity BreakdownCodeTaskDurationDependencies1Planning & Research6 Days (Week 4)None2Assembling Requirements6 Days (Week 5)13System Architecture Design12 Days (Weeks 6 & 7)1 & 24Prototype Development (Figma)24 Days (Weeks 7–10)2 & 35GitHub Documentation18 Days (Weeks 9–11)3 & 4 (Parallel)6Testing & Correction18 Days (Weeks 8–10)4 (Parallel)7Final Report Submission6 Days (Week 11)3, 4, 5, 6Critical Paths: 1 → 2 → 3 → 4 → 5 → 7 & 1 → 2 → 3 → 4 → 6 → 7 (Slack = 0 days).Risk Management Strategy: Mitigated scope creep in Task 4 by sticking to a frozen MVP feature set.
